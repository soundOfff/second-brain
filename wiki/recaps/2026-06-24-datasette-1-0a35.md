# Recap — Datasette 1.0a35 release

[[entities/simon-willison]] announces a "big" alpha release of
[[entities/datasette]] (1.0a35), his open-source multi-tool for exploring and
publishing data [2026-06-24-datasette-1-0a35].

## Key claims

- **Create table** UI in the database actions menu, backed by a `/<database>/-/create`
  JSON API: columns, primary keys, custom column types, NOT NULL constraints, literal
  and expression defaults, and single-column foreign keys (#2787)
  [2026-06-24-datasette-1-0a35].
- **Alter table** action and `/<database>/<table>/-/alter` JSON API: add/rename/
  reorder/drop columns; change types, defaults, NOT NULL, primary keys, foreign keys;
  rename or drop the table (#2788) [2026-06-24-datasette-1-0a35].
- **Template context documentation** for custom templates, treated as a **stable API
  until Datasette 2.0**. Generated from dataclass definitions next to the view code,
  with tests comparing documented fields against actually-rendered contexts (#1510,
  #2127, #1477, #2803) [2026-06-24-datasette-1-0a35].

Note: both new table features ship a JSON API alongside the UI — an instance of the
[[concepts/agent-native-infrastructure]] pattern (machine-readable surfaces beside the
human one).
